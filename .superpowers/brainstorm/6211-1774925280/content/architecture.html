Key Changes from Existing Architecture

Replacing (Rust)
  • Axum HTTP server
  • SQLx (compile-time SQL)
  • comrak, pdf-extract, zip+quick-xml
  • reqwest HTTP client for LLMs
  • Complex multi-stage Dockerfile
With (Python)
  • FastAPI + Uvicorn
  • SQLAlchemy + Alembic migrations
  • python-docx, PyPDF2, openpyxl, python-pptx
  • httpx async client for LLMs
  • Simple single-stage Dockerfile

What Stays The Same

  • React frontend — same SPA, same API contract, just points at Python instead of Rust
  • Next.js generator — themes untouched, still builds static sites
  • PostgreSQL schema — same tables (users, documents, portfolios, api_keys, sessions)
  • REST API contract — same endpoints, same request/response shapes
  • Docker Compose orchestration — same 4-service architecture
  • Portfolio JSON schema — the data structure spec is unchanged
